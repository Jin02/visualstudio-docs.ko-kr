# <a name="receive-activity-designer"></a>Receive 활동 디자이너
**수신** 활동 디자이너는 <xref:System.ServiceModel.Activities.Receive> 활동을 만들고 구성하는 데 사용됩니다. <xref:System.ServiceModel.Activities.Receive> 활동은 기본 제공 형식(예:  <xref:System.ServiceModel.Channels.Message>, <xref:System.IO.Stream> 또는 <xref:System.Xml.Linq.XElement>)이거나 serialize될 수 있는 애플리케이션 정의 데이터 계약, 메시지 계약 또는 XML 클래스 중 하나인 메시지를 수신하는 활동입니다.

## <a name="the-receive-activity"></a>Receive 활동
 <xref:System.ServiceModel.Activities.Receive> 활동은 사용된 수신 콘텐츠의 유형에 따라 단일 항목 또는 여러 항목을 받을 수 있습니다. <xref:System.ServiceModel.Activities.SendReply> 활동은 서비스에 대한 요청/응답 메시지 교환 패턴 중 메시지를 받는 <xref:System.ServiceModel.Activities.Receive> 활동에 바인딩될 수 있습니다.

### <a name="using-the-receive-activity-designer"></a>Receive 활동 디자이너 사용
 **활동 받기** 디자이너는 도구 **상자의** **메시징** 범주에서 찾을 수 있습니다.이 **도구 상자의 도구 상자** 탭을 [!INCLUDE[wfd2](../includes/wfd2-md.md)] 클릭 하 여 액세스할 수 있습니다 (또는 **보기** 메뉴에서 **도구 모음** 또는 CTRL+ALT+X를 선택 합니다.)

 **[활동 받기]** 디자이너는 **도구 상자에서** 끌어서 [!INCLUDE[wfd2](../includes/wfd2-md.md)] 활동이 배치되는 모든 위치에 놓일 수 있습니다. 그러면 기본 <xref:System.ServiceModel.Activities.Receive>인 Receive라는 이름의 <xref:System.Activities.Activity.DisplayName%2A> 활동이 만들어집니다. 수신 <xref:System.Activities.Activity.DisplayName%2A> 활동 디자이너의 헤더 **Receive** 또는 속성 그리드의 **DisplayName** 상자에서 편집할 수 있습니다.

 <xref:System.ServiceModel.Activities.SendReply> 활동을 <xref:System.ServiceModel.Activities.Receive> 만들고 선택한 활동에 바인딩하려면 활동 **수신** 디자이너를 마우스 오른쪽 단추로 클릭하고 컨텍스트 메뉴에서 **SendReply 만들기** 항목을 클릭하고 **SendReplyToReceive** 디자이너가 **수신** 디자이너 아래에 나타납니다. <xref:System.ServiceModel.Activities.SendReply> 활동은 서비스측에 대한 요청/응답 메시지 교환 패턴 중 응답 메시지를 보내는 활동입니다. **SendReplyToReceive** 디자이너로 구성할 수 있습니다.

 또는 **도구 상자의** **메시징** 범주에 있는 **ReceiveAndSendReply** 템플릿 디자이너를 사용하여 미리 구성된 활동 <xref:System.ServiceModel.Activities.Receive> <xref:System.ServiceModel.Activities.SendReply> 및 활동 쌍을 만들 수 있습니다. [!INCLUDE[crabout](../includes/crabout-md.md)]**ReceiveAndSendReply** 및 **SendReplyToReceive** 템플릿의 사용은 [ReceiveAndSendReply](../workflow-designer/receiveandsendreply-template-designer.md) 항목을 참조하십시오.
