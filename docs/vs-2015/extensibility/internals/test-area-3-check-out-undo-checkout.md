# <a name="test-area-3-check-outundo-checkout"></a>테스트 영역 3: 체크 아웃/체크 아웃 취소
[!INCLUDE[vs2017banner](../../includes/vs2017banner.md)]

이 소스 제어 플러그 인 테스트 영역에서는 통해 버전 저장소에서 항목을 편집 및 되돌리기 합니다 **체크 아웃** 및 **체크 아웃 취소** 명령입니다.  
  
 **체크 아웃**: 읽기/쓰기를 로컬 복사본을 수정 하는와 버전 저장소의 항목이 체크 아웃을 표시 합니다.  
  
 **체크 아웃 취소**: 표시 항목으로 버전 저장소에 체크 인 로컬 (옵션)에 따라 체크 아웃 하기 전에 상태 복사본을 되돌립니다.

### <a name="case-3b-disconnected-check-out"></a>사례 3b: 연결이 끊긴된 체크 아웃  
 연결이 끊어진된 모드에서 작동 일정 수준의 지속적인된 소스 제어 지원 버전 저장소에 직접 연결 되지 않은 경우 사용자가 있습니다. 참여 한 솔루션 및 프로젝트에 대 한 모든 관련 정보를 로컬에 캐시 하면 됩니다.  
  
 단독 체크 아웃 작업 소스 제어 저장소에 연결 하는 동안에 발생할 수 있습니다. 공유 체크 아웃 작업에 연결 되었거나 연결이 있는지 여부를 언제 든 지 발생할 수 있습니다. 따라서 버전 저장소 연결이 끊어졌을 때 합니다 **공유 아웃 확인** (CO) 명령을 사용할 수 있습니다. 연결이 끊긴 동안 **체크 아웃 취소** 사용할 수 없습니다. 사용자가 변경한 내용을 바꾸려면 이전 버전을 검색할 수 없습니다.  
  
 사용자 버전으로 다시 연결 될 때 모든 등록 된 솔루션의 체크 아웃 상태를 저장 및 프로젝트 동기화 됩니다. 이 사용자가 수행 하는 체크 아웃에 대 한 저장소에 필요한 업데이트를 수행 합니다. 동기화에 문제가 발생 한 후 사용자가 (연결) 정상적으로 작업을 계속할 수 있습니다.

- 사용할 수 없습니다 **체크 아웃 취소** 버전 저장소에서 연결이 끊어진 동안 명령입니다.

 **체크 아웃 실행 취소** 취소 파일의 체크 아웃 상태를 파일에 변경 내용을 체크 인하지 마십시오 하는 데 사용 됩니다.  
  
#### <a name="expected-behavior"></a>예상된 된 동작  
  
- 기본 사용자를 기반으로 **로컬 버전 체크 아웃** 설정 합니다. 사용자가 로컬 버전을 체크 아웃을 선택한 경우 항상 체크 아웃 한 버전으로 되돌리려면 체크 아웃을 취소에 대 한 기본값은입니다.  
  
- 실행 취소 아이콘의 동의 시 **솔루션 탐색기** 영향에 대 한 업데이트 된 파일 및 항목에서 제거 되는 **보류 중인 체크 인** 창입니다.  
  
|작업|테스트 단계|확인 하려면 예상된 결과|  
|------------|----------------|--------------------------------|  
|단독으로 체크 아웃 하는 단일 파일이 체크 아웃 취소|1.  클라이언트 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  파일을 배타적 체크 아웃 합니다.<br />4.  파일을 수정 합니다.<br />5.  체크 아웃 취소 (**파일**를 **소스 제어**하십시오 **체크 아웃 취소**).|일반적인 예상된 동작입니다.|  
|체크 아웃 공유 된 단일 파일의 체크 아웃 취소|1.  클라이언트 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  파일 공유를 확인 합니다.<br />4.  파일을 수정 합니다.<br />5.  체크 아웃 취소 (**파일**를 **소스 제어**하십시오 **체크 아웃 취소**).|일반적인 예상된 동작입니다.|  
|프로젝트에 파일을 추가한 후 프로젝트를 체크 아웃 취소|1.  새 프로젝트를 만들고 소스 제어에 추가 합니다.<br />2.  프로젝트를 확인 합니다.<br />3.  프로젝트에 파일을 추가 합니다.<br />4.  프로젝트의 체크 아웃을 취소 합니다.|추가 된 파일은 솔루션 탐색기에서 프로젝트에서 제거 됩니다.<br /><br /> 프로젝트 체크 아웃 하지 않았습니다.|  
|프로젝트에서 파일을 삭제 한 후 프로젝트를 체크 아웃 취소|1.  새 프로젝트를 만들고 소스 제어에 추가 합니다.<br />2.  프로젝트를 확인 합니다.<br />3.  프로젝트에서 파일을 삭제 합니다.<br />4.  프로젝트의 체크 아웃을 취소 합니다.|삭제 된 파일이 솔루션 탐색기의 프로젝트 아래에 나타납니다.<br /><br /> 프로젝트 체크 아웃 하지 않았습니다.|
