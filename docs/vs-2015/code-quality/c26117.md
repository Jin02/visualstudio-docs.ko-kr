# <a name="c26117"></a>C26117
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

C26117 경고: 잠금을 해제할 \<잠금 > 함수에서 \<func >.

구문상으로 범위가 지정 된 잠금 적용 *획득* 잠금이 *릴리스* 쌍 c에서 /C++ 프로그램 언어에서 수행 되지 않습니다. 함수가 현재 상태를 크게 수정함으로써 잠금 부작용이 발생할 수 있습니다. 예를 들어 잠금 래퍼 함수인 잠금 획득 또는 잠금 수를 지정 된 잠금에 대 한 개수를 증가 시킵니다. 부작용 잠금에서 획득 또는 잠금 해제를 사용 하 여 포함 된 함수에 주석을 달 수 있습니다 `_Acquires_lock_` 또는 `_Releases_lock_`, 각각. 이러한 주석 없이는 함수가 반환 후 잠금 수를 변경하지 않을 것으로 예상됩니다. 경우 획득 및 릴리스 균형이 맞지, 될 것으로 간주 되 *분리 된*합니다. 때로 주석이 추가 된가 있는 함수 경고 C26117 발생 `_Releases_lock_` 해제 하기 전에 함수가 잠금을 소유 해야 하기 때문에 잠금을 보유 하지를 해제 합니다.

## <a name="example"></a>예제
다음 예제에서는 때문에 C26117 경고를 생성 함수 `ReleaseUnheldLock` 유지 하지 해당 되는 잠금을 해제-상태의 `flag` 모호-않아야 함을 지정 하는 주석이 없는 및 합니다.

```cpp

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

int flag;

void ReleaseUnheldLock(DATA* p)
{
    if (flag)
        EnterCriticalSection(&p->cs);
    // code ...
    LeaveCriticalSection(&p->cs);
}
```

## <a name="example"></a>예제
다음 코드는 해제 된 잠금을 같은 조건에서 획득 하 여 문제를 해결 합니다.

```cpp

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

int flag;

void ReleaseUnheldLock(DATA* p)
{
    if (flag)
    {
        EnterCriticalSection(&p->cs);
        // code ...
        LeaveCriticalSection(&p->cs);
    }
}
```
