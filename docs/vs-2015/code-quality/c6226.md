# <a name="c6226"></a>C6226
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]
경고 C6226: 의미 체계가 다른 정수 형식 간의 암시적 캐스트: HRESULT에-1을 할당 합니다. E_FAIL을 대신 사용 하는 것이 좋습니다.

이 경고는 `HRESULT` 할당 또는-1의 명시적 값으로 초기화 합니다. 이 경고는 정수의 혼동 하 여 자주 발생 하 고 `HRESULT` 형식입니다. 성공을 표시 하기 위해, 기호화 된 상수를 사용 하 여 `S_OK` 대신 합니다. E_로 시작 하는 기호화 된 상수를 사용 하 여 실패를 나타내려면*상수*와 같은 `E_FAIL`합니다.

자세한 내용은 참조는 [SUCCEEDED](/windows/desktop/api/winerror/nf-winerror-succeeded) 및 [실패](/windows/desktop/api/winerror/nf-winerror-failed) 매크로입니다.

## <a name="example"></a>예제

다음 코드에서는이 경고를 생성합니다.

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = -1;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

이 경고를 해결 하려면 다음 코드를 사용 합니다.

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = E_FAIL;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

이 경고는 `SCODE` 형식으로 처리 됩니다는 `HRESULT`합니다.

사용 `malloc` 고 `free` (및 동적 메모리 Api 관련) 메모리 누수 및 예외를 발생 하는 많은 문제에 있습니다. 제공 하는 포인터와 컨테이너 클래스를 사용 하 여 이러한 종류의 누수 및 예외 문제를 방지 하려면는 C++ 표준 라이브러리입니다. 여기에 포함 됩니다 [shared_ptr](/cpp/standard-library/shared-ptr-class)하십시오 [unique_ptr](/cpp/standard-library/unique-ptr-class), 및 [벡터](/cpp/standard-library/vector). 자세한 내용은 [스마트 포인터](/cpp/cpp/smart-pointers-modern-cpp) 하 고 [ C++ 표준 라이브러리](/cpp/standard-library/cpp-standard-library-reference)합니다.
