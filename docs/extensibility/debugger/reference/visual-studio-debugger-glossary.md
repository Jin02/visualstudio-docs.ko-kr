# <a name="visual-studio-debugger-glossary"></a>Visual Studio 디버거 용어집
다음은 [!INCLUDE[vsprvs](../../../code-quality/includes/vsprvs_md.md)] 디버깅 SDK에 사용되는 용어입니다.

## <a name="terms"></a>용어
 바인딩된 중단점 코드에서 설정된 중단점에 대한 추상화입니다. 코드 스트림에는 바인딩된 중단점과 중단점 명령 사이에 일대일 관계가 있습니다. 코드가 언로드되면 바인딩된 중단점이 바인딩해제될 수 있습니다.

 인과 관계 여러 물리적 스레드, 프로세스 및 컴퓨터에서 실행의 논리적 스레드를 추적하고 해당 스레드의 수명 중 지정된 지점에서 해당 논리 스레드의 호출 스택을 재구성하는 기능을 제공합니다.

 코드 컨텍스트 디버그 엔진에 알려진 코드에서 위치의 추상화를 제공합니다. 대부분의 런타임 아키텍처에서 코드 컨텍스트는 프로그램의 명령 스트림의 주소입니다. 코드가 지침으로 표현되지 않을 수 있는 비전통적인 언어의 경우 코드 컨텍스트가 다른 수단으로 표현될 수 있습니다.

 코드 경로 분기가 수행되는 코드에서 실행 지점을 나타내거나 함수 호출이 수행됩니다. 스택 추적은 기본적으로 함수 호출 코드 경로의 목록입니다.

 디버그 엔진(DE) 런타임 아키텍처의 디버깅을 허용하는 구성 요소입니다. 디버그 엔진은 인터프리터 또는 운영 체제와 함께 작동하며 실행 제어, 중단점 및 식 평가와 같은 디버깅 서비스를 제공합니다.

 문서 컨텍스트 디버그 엔진에 알려진 원본 파일 문서의 위치 추상화를 제공합니다. 대부분의 언어에서 문서 컨텍스트는 원본 파일의 위치입니다. 원본 파일이 텍스트가 아닐 수 있는 비전통적인 언어의 경우 문서 컨텍스트가 다른 방법으로 표현될 수 있습니다. 문서 *위치도*참조하십시오.

 문서 위치 IDE에 알려진 소스 파일에서 위치의 추상화를 제공합니다. 대부분의 언어에서 문서 위치는 원본 파일의 위치입니다. 비전통적인 언어의 경우 문서 위치가 다른 방식으로 표시될 수 있습니다. 문서 *컨텍스트도*참조하십시오.

 오류 중단점 보류 중인 중단점에서 오류를 설명하는 추상화입니다. 오류 중단점은 보류 중인 중단점의 위치, 보류 중인 중단점과 연결된 식 또는 보류 중인 중단점이 코드 위치에 바인딩되지 않도록 하는 기타 정보를 설명할 수 있습니다.

 평가 컨텍스트 식 평가를 위한 프로그래밍 컨텍스트의 추상화를 제공합니다. 일반적으로 평가 컨텍스트는 범위입니다. 식 컨텍스트에서 식 평가를 수행할 때 식 컨텍스트는 생성 지점과 일치하는 범위 규칙을 제공합니다. 예를 들어 스택 프레임에서 만든 식 컨텍스트는 지역 변수, 메서드 매개 변수, 클래스 멤버(해당하는 경우) 및 전역 변수를 평가하기 위한 컨텍스트를 제공합니다.

 가로채는 예외 현재 스택 프레임에 예외 처리 메커니즘이 없는 경우에도 디버그 엔진에 의해 가로채는 예외입니다.

 JustMyCode 사용자에 속한 코드만 디버깅하고 해당 시스템 코드에 대해 소스 코드를 사용할 수 있는 경우에도 시스템 코드와 같은 모든 중간 코드를 무시하는 개념입니다.

 보류 중인 중단점은 코드가 로드되기 전, 도중 및 후에 중단점에 대한 추상화를 제공하고 중단점을 가상화하는 방법을 제공합니다. 보류 중인 중단점:

- 하나 이상의 프로그램에서 코드에 중단점을 바인딩하는 데 필요한 모든 정보를 포함합니다.

- 하나 이상의 프로그램에서 여러 코드 위치에 바인딩할 수 있습니다.

- 절대로 코드에 자신을 바인딩하지 마십시오.

  코드가 로드될 때마다 프로그램의 모든 보류 중인 중단점이 바인딩할 수 있는지 확인합니다. 보류 중인 중단점은 바인딩되는 모든 바인딩된 중단점을 포함한다고 합니다.

  물리적 Win32 프로세스를 처리합니다. 프로세스에 여러 프로그램이 포함될 수 있습니다. *프로그램*도 참조하십시오.

  프로그램 특정 런타임 아키텍처 내에서 실행되는 단일 네임스페이스입니다. 또한 *프로세스를*참조하십시오.

  세션 디버그 관리자(SDM) 여러 컴퓨터에서 여러 프로세스의 프로그램 수를 디버깅하는 여러 수의 디버그 엔진을 관리합니다. 기본 수준에서 SDM은 디버그 엔진의 멀티플렉서입니다. 또한 SDM은 IDE에 디버깅 세션의 통합보기를 제공합니다.

  스택 프레임은 특정 프레임의 계산 상태와 중첩된 함수 호출의 특정 수준을 나타냅니다.

  스레드 하나 이상의 프로그램에서 실행되는 스택 기반 명령 실행의 일반화 개념입니다.

  경고 중단점 보류 중인 중단점에서 경고를 설명하기 위한 추상화입니다. 경고 중단점은 보류 중인 중단점이 아직 코드 위치에 바인딩되지 않은 이유를 설명합니다. 보류 중인 중단점에 의해 설명된 위치에 대해 코드가 아직 로드되지 않았거나 다른 이유로 인해 로드되지 않았을 수 있습니다.
