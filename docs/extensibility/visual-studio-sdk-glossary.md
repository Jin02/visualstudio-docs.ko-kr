# <a name="visual-studio-sdk-glossary"></a>비주얼 스튜디오 SDK 용어집
이 용어는 설명서에 사용되는 용어에 [!INCLUDE[vsipsdk](../extensibility/includes/vsipsdk_md.md)] 대한 정의를 제공합니다.

## <a name="terms"></a>용어
 추가 기능 유틸리티 응용 프로그램, 드라이버 또는 기본 응용 프로그램에 추가된 기타 소프트웨어입니다. Visual Studio 통합 개발 환경(IDE)에서 추가 기능은 IDE의 기능을 확장하는 자동화 기반 응용 프로그램입니다.

 자동화 모델 이전 버전의 Visual Studio에서 확장성 모델로 알려진 자동화 모델은 IDE를 구동하는 기본 루틴에 액세스할 수 있는 프로그래밍 인터페이스입니다. 추가 기능, 마법사 및 매크로는 자동화 모델의 개체를 사용하여 IDE의 기능을 제어하거나 확장합니다.

 명령 UI 컨텍스트 도구 모음과 같은 UI 명령 또는 요소의 가시성과 GUID의 연결입니다. UI 명령 컨텍스트는 창에 연결되지 않는다는 점에서 선택 컨텍스트와 는 다릅니다.

 명령 UI 컨텍스트를 사용하여 다음을 수행할 수 있습니다.

- 특정 창이 활성화될 때 나타나는 도구 모음에 GUID를 할당합니다.

- VSPackage를 로드하거나 실행할 필요 없이 명령의 가용성에 GUID를 할당합니다.

- 활성 키 바인딩에 영향을 주는 GUID를 할당합니다.

- 매크로 기록을 켜려면 GUID를 할당합니다.

- 디버그 모드를 활성화하거나 편집기에서 디자인 과 실행 모드 사이를 전환하는 GUID를 할당합니다.

  소프트웨어 구현에 대한 기존 정보가 없는 응용 프로그램 없이 응용 프로그램의 기능의 일부로 만들 수 있는 구성 요소 소프트웨어 조각입니다. 구성 요소와 응용 프로그램 간의 통신은 전적으로 OLE 스타일 인터페이스를 통해서만 이루어집니다.

  구성 요소 관리자 `SOleComponentManager`최상위 구성 요소에 대 한 비 사용자 인터페이스 조정 서비스를 제공 하는 서비스 입니다. 서비스는 `SOleComponentManager` 인터페이스를 `IOleComponentManager` 구현합니다.

  구성 요소 UI `SOleComponentUIManager`관리자 사용자 인터페이스 조정 서비스를 제공 하는 서비스 입니다. 서비스는 `SOleComponentUIManager` `IOleComponentUIManager` 및 `IOleInPlaceComponentUIManager` 인터페이스를 구현합니다.

  컨텍스트 백 `IVsUserContext` 환경 구성 요소에 연결된 개체(COM 개체)입니다. 이 개체에는 조회 키워드, **F1** 키워드 및 구성 요소와 관련된 특성이 포함됩니다. 컨텍스트 백은 또한 연결된 모든 컨텍스트 백을 가리킵니다.

  컨텍스트 공급자와 연결된 컨텍스트 백이 있는 IDE의 구성 요소입니다. 이러한 구성 요소에는 도구 창, 편집기 또는 프로젝트 계층 구조가 포함됩니다.

  디자이너 사용자가 UI의 요소(양식, 단추 및 기타 컨트롤)를 조작할 수 있는 프로그래밍 인터페이스입니다.

  DocData 문서/뷰 구분이 있는 세계에서 문서의 기본 데이터를 캡슐화하는 COM 개체입니다(예: 텍스트 편집기의 경우 모든 텍스트 편집기 뷰의 기본 텍스트 버퍼임). EditorFactory가 이 개체를 제공하지 않는 경우 IDE는 해당 개체를 대신 합니다. 이 개체의 책임은 동일한 `DocData`에 대한 여러 뷰에 대한 데이터 지속성 및 공유 의미 체계를 관리하는 것입니다. 개체가 `DocData` 인터페이스를 `IOleCommandTarget` 지원하는 경우 UIShell의 명령 라우팅에 포함됩니다.

  DocObject 기술은 호스트가 제공하는 프레임 내에서 UI를 호스트하는 데 사용됩니다. 보다 구체적으로 이 용어는 `IOleDocument` 인터페이스 및 관련 인터페이스를 지원하는 모든 포함을 의미합니다. 이 기술에는 COM 문서의 구현 세부 정보, Visual Basic 5.0의 도구 창, Visual Basic 6.0의 ActiveX 디자이너 등과 같은 많은 잠재적 응용 프로그램이 있습니다.

  문서 를 전체적으로 참조하는 데 사용됩니다. `DocData` `DocView` 예를 들어 DocumentFrame에는 `DocView`을 포함하지만 지속성을 처리하는 `DocData` 에 대한 참조도 유지됩니다.

  DocView 사용자가 상호 작용하여 기본 `DocData`을 보고 조작하는 DocObject/포함/WindowPane입니다. 사용자는 인터페이스 디자인의 일부인 문서/보기 분리를 `DocObject` 활용하지 않습니다. 사용자는 전체 DocObject를 사용하여 로 알려진 기본 데이터의 보다 추상적이고 덜 공식화된 `DocData`개념을 사용하는 대신 뷰 역할을 합니다. `DocView`개체는 항상 IDE의 문서 프레임 개체(MDI 자식 창)와 함께 포함됩니다.

  `DTE` DTE(개발 도구 확장성) 개체는 Visual Studio 자동화 모델에서 가장 높은 액세스 지점으로, 프로그래밍 방식으로 IDE를 자동화하고 확장할 수 있습니다.

  IDE에서 구현되고 조회 키워드 또는 **F1** 도움말 항목 목록을 표시하는 동적 도움말 창 창입니다.

  을 구현하는 편집기 코드(클래스, `DocView`CLSID)는 또한 뷰 `DocData` 및 데이터 분리가 지원되는 경우에도 구현합니다.

  확장 IDE에 수정, 사용자 지정 또는 추가하는 기능입니다. 자동화 모델 또는 VSPackage를 사용하여 확장을 만듭니다.

  외부 편집기 마이크로소프트 워드와 같은 IDE에 국한되지 않는 편집기입니다. 자체 메커니즘을 통해 등록되었으며 IDE 외부에서 사용할 수 있습니다. 이 편집기는 포함할 수 있는 경우 IDE의 창 내에 나타납니다. 포함할 수 없는 경우 별도의 최상위 창이 만들어집니다.

  계층 구조 노드 트리( 속성 집합과 연결된 각 노드)

  독립적인 최상위 구성 요소 A 구성 요소는 모덜리스 최상위 창을 사용하고 독립 실행형 응용 프로그램 창으로 효과적으로 작동할 수 있지만 프로세스 내 개체로 구현됩니다. 따라서 독립적인 최상위 구성 요소는 양식 및 메시지 루프 서비스를 IDE와 조정해야 합니다. 프로세스 내 개체에는 고유한 메시지 루프가 없습니다.

  정보 공급자 정보 공급자는 키워드를 조회하고 `IVsUserContextItem` 개체 의 형태로 주제 목록을 반환할 수 있는 모듈입니다. 정보 공급자에 대한 **F1** 및 조회 키워드 항목을 제공하려면 컴파일된 도움말 파일(를*등록합니다. HxS)를*시스템과 함께. 이 파일의 도움말 항목은 동적 도움말 창에 표시되는 항목 목록을 제공하고 사용자가 **F1을**누르는지 여부를 표시합니다.

  IDE가 소유한 문서 창 내에 `IOleInPlaceComponent` 시각적으로 포함된 창을 관리하기 위해 인터페이스를 구현하는 내부 구성 요소 A VSPackage 개체입니다. 현재 구성 요소는 표준 OLE 메뉴 병합에 참여하지 않습니다. 대신 사용자 인터페이스 요소를 IDE에 통합합니다.

  내부 구성 요소에는 유선 부품과 구성 요소 컨트롤의 두 가지 유형이 있습니다.

  유선 된 사내 구성 요소에는 IDE의 사용자 인터페이스에 단단히 통합 된 메뉴, 도구 모음 및 명령이 있으며 IDE에 직접 내장 된 것처럼 나타납니다.

  구성 요소 컨트롤에는 IDE에 통합된 자체 사용자 인터페이스 요소가 없습니다. 대신 IDE의 메뉴, 명령 및 도구 모음을 사용합니다. 예를 들어 굵게 명령을 사용하여 양식에 포함된 리치 텍스트 컨트롤 내에서 선택한 단어를 굵게 할 수 있습니다. 그러나 구성 요소 컨트롤은 동적으로 설치된 구성 요소별 UI 요소를 표시해 달라고 요청할 수 있습니다.

  언어 서비스 VSPackage 개발자가 텍스트 표시 및 색지정과 같은 컴퓨터 언어 코드 편집기의 기능을 구현할 수 있는 개체 집합입니다.

  기타 파일 프로젝트 프로젝트는 어떤 프로젝트에없는 열려있는 파일을 수용하는 데 사용됩니다. 이 프로젝트의 항목 목록이 유지되지 않습니다.

  프로젝트 프로젝트는 계층 구조 개체 또는 인터페이스를 `IVsHierarchy` 구현하는 COM 개체로 구성됩니다.

  프로젝트 유형과 독립적으로 사용할 수 없는 프로젝트별 디자이너 또는 편집기 디자이너입니다. 모든 프로젝트별 디자이너는 레지스트리에 편집기 공장 정보를 입력해야 합니다. 그런 다음 IDE는 특정 프로젝트에서 특정 파일 형식이 열릴 때마다 디자이너를 인스턴스화할 수 있습니다.

  프로젝트 유형 창 전역 선택 컨텍스트에서 현재 활성 프로젝트 계층 및 항목을 지속적으로 추적하는 창입니다. 프로젝트 유형 창은 `SVsTrackSelectionEx` 서비스를 사용하여 IDE에 변경 사항을 알리고 사용자에게 피드백을 표시합니다. 솔루션 탐색기는 프로젝트 유형 창의 예입니다.

  속성 창 이전 속성 브라우저입니다.

  참조 기반 프로젝트 프로젝트의 파일이 동일한 디렉터리에 있어야 하지 않는 프로젝트입니다. 대신 다른 관련없는 디렉터리에서 파일에 대한 참조가 프로젝트 자체에 의해 저장되고 유지 관리됩니다.

  실행 중인 문서 테이블 IDE가 현재 열려 있는 모든 문서의 목록을 유지 관리하는 내부 구조입니다. 목록에는 문서가 현재 편집중인지 여부에 관계없이 메모리에 열려 있는 모든 문서가 포함됩니다. 문서는 편집기에서 열린 저장 프로시저, 프로젝트의 파일 또는 기본 프로젝트 파일(예: *.vcproj 파일)을 포함하여 저장된 모든 항목입니다.

  선택 컨텍스트 IDE의 모든 창의 세부 정보의 일부이며 활성 선택을 추적하는 데 사용되는 데이터입니다. 선택 컨텍스트는 다음으로 구성됩니다.

- 프로젝트 계층 `IVsHierarchy` 구조의 인터페이스에 대한 포인터

- 프로젝트 항목의 항목 식별자입니다.

- 활성 개체에 `ISelectionContainer` 대한 속성에 대한 액세스를 제공하는 인터페이스에 대한 포인터입니다.

- 요소 값의 배열입니다.

  service 단일 COM 개체에 있는 COM 인터페이스 집합에 대한 계약입니다. GUID로 식별되는 서비스를 만들 때 서비스를 수행하는 COM 인터페이스 집합을 정의합니다. COM 개체는 서비스를 사용하여 서로 통신합니다.

  사용자가 작업하는 관련 프로젝트의 솔루션 그룹입니다.

  표준 디자이너 프로젝트 유형과 독립적으로 사용할 수 있는 디자이너입니다. 모든 표준 디자이너는 레지스트리에 편집기 공장 정보를 입력해야 합니다. 그런 다음 IDE는 특정 확장자가 있는 파일이 열릴 때마다 디자이너를 인스턴스화할 수 있습니다. 데이터는 파일에 유지되어야 합니다.

  표준 편집기 는 특정 프로젝트 유형과 독립적으로 사용할 수 있습니다. 이러한 편집자에는 레지스트리에 등록된 Editor공장이 있습니다. 이렇게 하면 IDE가 편집기를 찾아 호출할 수 있습니다.

  표준 OS 편집기 비주얼 스튜디오 특정되지 않은 포함. 잘 알려진 Win32 키를 사용하여 등록됩니다(예: Win32 탐색기는 호출 하는 방법을 알고 있음). 이러한 편집기는 포함할 수 있는 경우 편집기는 여전히 IDE의 위치에 표시됩니다. 그렇지 않으면 이러한 편집기에 대해 별도의 최상위 창이 만들어집니다.

  컨텍스트 백 `IVsUserContext` 컨텍스트 가방에 연결된 개체입니다. 개체는 조회 키워드, **F1** 키워드 및 IDE 구성 요소 내에서 선택 영역에 대 한 속성을 보유 합니다. 하위 컨텍스트의 예로는 도구 창의 명령 또는 편집기의 키워드가 있습니다.

  IDE에서 구현하고 활성 작업 목록을 표시하는 작업 목록 도구 창입니다.

  텍스트 버퍼 개체의 `VSTextBuffer`일반 이름 .

  텍스트 보기 개체의 `VSTextView`공통 이름 .

  도구 최상위 구성 요소 모덜리스 팝업 창으로 작동 하며 IDE의 사용자 인터페이스와 긴밀하게 조정됩니다. 독립적인 최상위 구성 요소와 마찬가지로 도구 최상위 구성 요소도 IDE와 양식 및 메시지 루프 서비스를 조정해야 합니다.

  최상위 구성 요소 A VSPackage 개체는 IDE 창의 클라이언트 영역 대신 모덜리스 최상위 창을 관리합니다. 최상위 구성 요소는 `IOleComponent` 유휴 시간에 대한 액세스와 같은 메시지 루프 서비스를 활용하기 위해 인터페이스를 구현합니다.

  UI 활성 VSPackage 개체가 표시되고 현재 포커스가 있습니다.

  계층 구조를 표시할 수 `IVsUIHierarchy` 있도록 인터페이스를 구현하는 UI 계층 구조 A COM 개체입니다. UI 계층 구조 창은 속성 창을 업데이트하는 `ISelectionContainer` 인터페이스를 구현합니다. 원하는 경우 이 구현을 사용할 수 있습니다.

  VSCT 비주얼 스튜디오 명령 테이블입니다. .vsct 파일에는 XML 형식의 메뉴, 도구 모음 및 명령의 배치 및 동작에 대한 정보가 포함되어 있습니다.

  VSPackage 사용자 인터페이스, 서비스, 프로젝트 유형 또는 편집기/디자이너 중 하나 이상을 기여하여 Visual Studio IDE를 확장하는 설치 가능한 소프트웨어입니다. VSPackage는 `IVsPackage` 인터페이스를 구현하는 COM 개체와 선택 및 기타 기능을 지원하기 위해 다른 인터페이스를 구현하는 하나 이상의 다른 COM 개체로 구성됩니다. 또한 VSPackage에는 특정 등록 요구 사항이 있습니다.
