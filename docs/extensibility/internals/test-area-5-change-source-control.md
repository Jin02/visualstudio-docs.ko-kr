# <a name="test-area-5-change-source-control"></a>테스트 영역 5: 소스 제어 변경
이 소스 제어 플러그인 테스트 영역은 **소스 제어 변경** 명령을 통해 소스 제어 변경을 다룹니다.

 **소스 제어 변경** 명령은 사용자에게 다음 네 가지 기본 기능을 제공합니다.

- **바인딩할:**

   사용자가 솔루션/프로젝트와 버전 저장소 간에 소스 제어 링크를 설정하거나 다시 설정할 수 있습니다.

- **언바인딩:**

   연결별로 소스 제어에서 프로젝트/솔루션을 제거합니다.

- **연결/연결 해제:**

  영역 3에서 다루는 제어 솔루션의 연결 또는 오프라인 상태를 전환합니다. 자세한 내용은 [테스트 영역 3: 체크 아웃/취소 체크 아웃](../../extensibility/internals/test-area-3-check-out-undo-checkout.md)을 참조하십시오.

## <a name="command-menu-access"></a>명령 메뉴 액세스
 다음 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 통합 개발 환경 메뉴 경로는 테스트 사례에 사용됩니다.

 소스 제어 변경:**파일,** **소스 제어,** **소스 제어 변경.**

## <a name="test-cases"></a>테스트 사례
 다음은 **변경 소스 제어** 명령 테스트 영역에 대한 특정 테스트 사례입니다.

### <a name="case-5a-bind"></a>케이스 5a: 바인드
 바인드를 사용하면 선택한 프로젝트 및 솔루션에 소스 코드 제어 정보를 추가할 수 있습니다. 사용자는 일반적으로 이러한 프로젝트를 추가할 소스 제어에서 프로젝트를 식별하라는 메시지가 표시됩니다. 사용자는 이 작업의 일부로 소스 컨트롤에서 새 프로젝트를 만들 수 없습니다(소스 컨트롤에 추가와 는 대조적).

| 작업 | 테스트 단계 | 확인할 예상 결과 |
| - | - | - |
| 빈 위치에 바인딩 | 1. 프로젝트를 만듭니다.<br />2. 소스 제어에 솔루션을 추가합니다.<br />3. 오픈 **변경 소스 제어** 대화 상자 **(파일,** **소스 제어,** **변경 소스 제어).**<br />4. **바인드 해제를 클릭합니다.**<br />5. 경고 대화 상자가 나타나면 수락합니다.<br />6. 모든 항목을 선택합니다.<br />7. **바인드**.<br />8. 소스 제어 저장소의 빈 위치로 이동합니다.<br />9. **확인을** 클릭하여 **소스 제어** 변경 대화 상자를 닫습니다.<br />10. 확인 대화 상자에서 **이러한 바인딩을 계속을 클릭합니다.**<br />11. 경고 대화 상자에서 **확인을** 클릭합니다.<br />12. 모든 것을 체크인합니다. 이 단계가 성공하면 다음 단계로 계속합니다.<br />13. 소스 제어에서 새 위치로 솔루션을 엽니다. | `Result from Step 12:`<br /><br /> 솔루션 및 프로젝트는 버전 저장소의 새 대상에 바인딩되고 기록됩니다.<br /><br /> 솔루션 및 프로젝트 파일이 체크 인됩니다.<br /><br /> 버전 저장소 프로젝트 계층은 디스크에 있는 프로젝트의 폴더 계층 구조를 일치시다.<br /><br /> `Result from Step 13:`<br /><br /> 모든 프로젝트 항목이 다운로드됩니다. |
| 클라이언트와 동기화된 위치에 바인딩 | 1. 프로젝트를 만듭니다.<br />2. 소스 제어에 솔루션을 추가합니다.<br />3. 버전 저장소에서 솔루션 및 프로젝트의 복제본을 만듭니다(사용 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]하는 경우 공유 및 분기).<br />4. 오픈 **변경 소스 제어** 대화 상자 **(파일,** **소스 제어,** **변경 소스 제어).**<br />5. 모두 바인딩 해제합니다.<br />6. **확인을** 클릭하여 **소스 제어** 변경 대화 상자를 닫습니다.<br />7. **변경 소스 제어** 대화 상자를 다시 엽니다.<br />8. 모두 선택합니다.<br />9. **바인드**.<br />10. 솔루션 및 프로젝트의 분기 위치로 이동(3단계에서)<br />11. **확인을** 클릭하여 **소스 제어** 변경 대화 상자를 닫습니다.<br />12. 모든 항목에 대한 최신 재귀 가져옵니다. | get 후 파일 콘텐츠는 get 이전과 동일합니다. |
| 클라이언트와 동기화되지 않은 위치에 바인딩 | 1. 프로젝트를 만듭니다.<br />2. 소스 제어에 솔루션을 추가합니다.<br />3. 버전 저장소에서 솔루션 및 프로젝트의 복제본을 만듭니다(사용 [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)]하는 경우 공유 및 분기).<br />4. 버전 저장소의 분기 된 프로젝트에서 파일을 수정합니다.<br />5. 오픈 **변경 소스 제어** 대화 상자 **(파일,** **소스 제어,** **변경 소스 제어).**<br />6. 모든 바인딩을 해제합니다.<br />7. **확인을** 클릭하여 **소스 제어** 변경 대화 상자를 닫습니다.<br />8. **변경 소스 제어** 대화 상자를 다시 엽니다.<br />9. 모두 를 선택합니다.<br />10. **바인드**.<br />11. 솔루션 및 프로젝트에 대한 분기 위치로 이동합니다.<br />12. **확인을** 클릭하여 **소스 제어** 변경 대화 상자를 닫습니다.<br />13. 경고 대화 상자가 나타나면 수락합니다.<br />14. 모든 항목에 대한 최신 재귀 받기. | 4단계에서 수정된 파일도 로컬에서 수정됩니다. |
| 소스 제어를 받지 않은 바인드 솔루션 | 1. 소스 컨트롤에서 빈 폴더를 만듭니다.<br />2. 클라이언트 프로젝트를 만듭니다.<br />3. 오픈 **변경 소스 제어** 대화 상자 **(파일,** **소스 제어,** **변경 소스 제어).**<br />4. 소스 제어의 빈 위치에 솔루션을 바인딩합니다.<br />5. **확인을** 클릭하여 **소스 제어** 변경 대화 상자를 닫습니다.<br />6. 확인 대화 상자에서 **이러한 바인딩을 계속을 클릭합니다.**<br />7. 경고 대화 상자에서 **확인을** 클릭합니다. | 솔루션이 소스 제어에 추가됩니다.<br /><br /> 솔루션 및 프로젝트가 체크 아웃되었습니다. |
| 바인드 취소 | 1. 프로젝트를 만듭니다.<br />2. 소스 제어에 솔루션을 추가합니다.<br />3. 변경 소스 제어 대화 상자를 엽니다.<br />4. 모두 바인딩 해제합니다.<br />5. **확인** 버튼을 클릭하여 대화 상자를 닫습니다. 이 단계가 성공하면 다음 단계로 계속합니다.<br />6. 변경 **소스 제어** 대화 상자를 다시 엽니다.<br />7. 관련없는 위치에 바인딩합니다.<br />8. **취소를 클릭합니다.** | `Result from Step 5:`<br /><br /> 솔루션이 더 이상 소스 제어를 받지 않습니다.<br /><br /> `Result from Step 8:`<br /><br /> 솔루션은 여전히 소스 제어하에 있지 않습니다. |

### <a name="case-5b-unbind"></a>케이스 5b: 언빈드
 Unbind는 프로젝트 및 해당 솔루션에서 소스 코드 제어 정보를 제거합니다. 영향을 받는 프로젝트와 솔루션은 사용자 선택과 소스 제어에 항목이 추가된 방법을 혼합하여 기반으로 합니다.

|작업|테스트 단계|확인할 예상 결과|
|------------|----------------|--------------------------------|
|하나의 파일 시스템 또는 로컬 IIS 웹 프로젝트와 하나의 클라이언트 프로젝트를 포함하는 바인딩 해제 솔루션|1. 파일 시스템 또는 로컬 IIS 웹 프로젝트를 만듭니다.<br />2. 소스 제어에 솔루션을 추가합니다.<br />3. 솔루션에 새 클라이언트 프로젝트를 추가합니다.<br />4. 메시지가 표시되면 솔루션 체크 아웃을 수락합니다.<br />5. **변경 소스 제어** 대화 상자를 엽니다.<br />6. **바인드 해제를 클릭합니다.**<br />7. **확인을** 클릭하여 대화 상자를 닫습니다.<br />8. 솔루션, 프로젝트, 솔루션 항목, 프로젝트 항목을 체크 아웃하려고 시도합니다.|솔루션 및 프로젝트는 소스 제어를 받지 않습니다.<br /><br /> 소스 제어 메뉴 명령이 나타나지 않습니다.|
|바인딩 해제 취소|1. 프로젝트를 만듭니다.<br />2. 소스 제어에 솔루션을 추가합니다.<br />3. **변경 소스 제어** 대화 상자를 엽니다.<br />4. **모든 바인딩 해제를 클릭합니다.**<br />5. **취소를**클릭합니다.|솔루션은 소스 제어를 받고 있습니다.|

### <a name="case-5c-rebind"></a>케이스 5c: 리바인드
 Rebind는 단순히 unbind 및 바인딩-소스 제어 이전에 소스 제어 하 고 바인딩되지 않은 프로젝트/솔루션을 다시 바인딩하는 프로세스의 조합입니다.

|작업|테스트 단계|확인할 예상 결과|
|------------|----------------|--------------------------------|
|**변경 소스 제어** 대화 상자를 닫지 않고 솔루션 및 프로젝트 다시 바인딩|1. 프로젝트를 만듭니다.<br />2. 소스 제어에 솔루션을 추가합니다.<br />3. **변경 소스 제어** 대화 상자를 엽니다.<br />4. **바인드 해제를 클릭합니다.**<br />5. 모든 행을 선택합니다.<br />6. **바인드**.<br />7. **확인을** 클릭하여 **소스 제어** 변경 대화 상자를 닫습니다.<br />8. 메시지가 표시되면 체크아웃을 수락합니다.|솔루션 및 프로젝트는 소스 제어하에 있습니다.|
|**변경 소스 제어** 대화 상자를 닫지 않고 프로젝트 Rebind|1. 프로젝트를 만듭니다.<br />2. (파일 >소스 제어 >소스 제어에 선택한 프로젝트를 추가하여 소스 제어에 프로젝트만 추가합니다.<br />3. 변경 소스 제어 대화 상자를 엽니다.<br />4. 프로젝트만 바인딩 해제합니다.<br />5. 프로젝트만 바인딩합니다.|솔루션은 제어되지 않은 상태로 유지됩니다.<br /><br /> 프로젝트는 제어된 상태로 유지됩니다.|
|**변경 소스 제어** 대화 상자를 닫지 않고 만 솔루션을 다시 바인딩|1. 프로젝트를 만듭니다.<br />2.**(파일,** **소스 제어,** 소스 제어에 선택한 프로젝트를 추가)를 사용하여 소스 제어에 솔루션만 **추가합니다.**<br />3. **변경 소스 제어** 대화 상자를 엽니다.<br />4. 솔루션만 바인딩 **해제(변경 소스 제어** 대화 상자를 닫지 마십시오.)<br />5. 용액만 바인딩합니다.<br />6. **확인을** 클릭하여 대화 상자를 닫습니다.<br />7. 솔루션 및 솔루션 항목 확인(있는 경우)|솔루션은 제어된 상태로 유지됩니다.<br /><br /> 프로젝트는 제어되지 않은 상태로 유지됩니다.|
|동일한 디렉터리에서만 솔루션/프로젝트 다시 바인딩|1. 프로젝트를 만듭니다.<br />2.**(파일,** **소스 제어,** 소스 제어에 선택한 프로젝트를 추가)를 사용하여 소스 제어에 프로젝트만 **추가합니다.**<br />3. 용액을 닫습니다.<br />4. 두 개 이상의 프로젝트로 새 솔루션을 만듭니다.<br />5. 소스 제어에 솔루션을 추가합니다.<br />6. 소스 제어에서 1단계에서 만든 프로젝트를 추가합니다.<br />7. 메시지가 표시되면 솔루션의 체크 아웃을 수락합니다.<br />8. 전체 솔루션을 확인합니다.<br />9. **변경 소스 제어** 대화 상자를 엽니다.<br />10. 추가된 프로젝트(6단계에서)를 선택하고 **바인딩 해제를 클릭합니다.**<br />11. **확인을** 클릭하여 대화 상자를 닫습니다.<br />12. 메시지가 표시되면 체크 아웃을 수락합니다.<br />13. **변경 소스 제어** 대화 상자를 다시 엽니다.<br />14. 추가된 프로젝트(6단계에서)를 선택하고 **바인딩을**클릭합니다.<br />15. 원래 위치를 선택합니다.|솔루션 및 프로젝트는 제어된 상태로 유지됩니다.|
