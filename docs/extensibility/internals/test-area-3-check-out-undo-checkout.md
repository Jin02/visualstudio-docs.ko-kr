# <a name="test-area-3-check-outundo-checkout"></a>시험 영역 3: 체크 아웃/취소 체크아웃
이 소스 제어 플러그인 테스트 영역은 **체크 아웃** 및 체크 **아웃 취소** 명령을 통해 버전 저장소에서 항목을 편집하고 되돌리는 것을 다룹니다.

**체크 아웃**: 버전 저장소에 항목을 체크 아웃된 것으로 표시하고 로컬 복사본을 읽고 쓰기로 수정합니다.

**체크 아웃 취소**: 버전 저장소의 항목을 체크 인된 상태로 표시하고 체크 아웃 하기 전에 로컬 복사본을 상태로 되돌리시고(옵션에 따라 다름).

## <a name="command-menu-access"></a>명령 메뉴 액세스

다음 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] 통합 개발 환경 메뉴 경로는 테스트 사례에 사용됩니다.

##### <a name="check-out"></a>체크 아웃:

- **파일**, **소스 제어**, **체크 아웃**.

- **파일,** **체크 아웃**.

- 바로 가기 메뉴, **체크 아웃**.

- 체크 아웃 취소: **파일,** **소스 제어,** **취소 체크 아웃 .**

## <a name="common-expected-behavior"></a>일반적인 예상 동작

- 체크 아웃 작업이 끝나면 대상 파일 및/또는 폴더가 버전 저장소에서 체크 아웃된 것으로 표시됩니다.

- 버전 저장소는 올바른 사용자에게 체크 아웃을 특성으로 합니다.

- 사용자의 설정에 따라 체크 아웃 의 시간과 날짜가 정확합니다.

### <a name="case-3b-disconnected-check-out"></a>사례 3b: 연결 해제 된 체크 아웃

연결이 끊긴 모드에서 작동하면 버전 저장소에 직접 연결되지 않은 경우 일정 수준의 지속적인 소스 제어 지원을 사용할 수 있습니다. 이 작업은 등록된 솔루션 및 프로젝트에 대한 모든 관련 정보를 로컬로 캐싱하여 수행됩니다.

단독 체크 아웃 작업은 소스 제어 저장소에 연결된 상태에서만 발생할 수 있습니다. 공유 체크 아웃 작업은 연결 또는 연결 해제 여부에 관계없이 언제든지 발생할 수 있습니다. 따라서 버전 저장소에서 연결이 끊어지면 COS(체크 **아웃 공유)** 명령만 활성화됩니다. 연결이 끊어지는 동안 이전 버전을 검색하여 사용자가 변경한 내용을 대체할 수 없으므로 **수행 취소 체크 아웃이** 비활성화됩니다.

사용자가 버전 저장소에 다시 연결하면 모든 등록된 솔루션 및 프로젝트의 체크 아웃 상태가 동기화됩니다. 이렇게 하면 사용자가 수행한 체크 아웃에 대해 저장소에 필요한 업데이트가 수행됩니다. 동기화가 완료되면 사용자는 정상(연결됨)으로 작업을 계속할 수 있습니다.

#### <a name="expected-behavior"></a>예상되는 동작

- 버전 저장소에서 연결이 끊어진 상태에서는 **체크 아웃 명령을 단독으로** 사용할 수 없습니다.

- 버전 저장소에서 연결이 끊어진 상태에서는 **검사 취소** 명령을 사용할 수 없습니다.

- **공유 체크 아웃** 명령이 작동합니다.

|작업|테스트 단계|확인할 예상 결과|
|------------|----------------|--------------------------------|
|연결이 끊긴 동안 파일을 체크 아웃한 다음 동기화를 위해 연결합니다.|1. 변경 소스 제어 대화 상자 **(파일,** **소스 제어,** **소스 제어 변경)를**사용하여 제어 된 프로젝트의 연결을 끊습니다.<br />2. 파일을 체크 아웃합니다.<br />3. 경고 대화 상자에서 체크 아웃(연결이 끊긴)을 클릭합니다.<br />4. 파일을 편집합니다.<br />5. 소스 제어 변경 대화 상자를 사용하여 연결합니다.<br />6. 편집 된 파일의 최신 버전을 가져옵니다.|일반적인 예상 동작|

### <a name="case-3c-query-editquery-save-qeqs"></a>사례 3c: 쿼리 편집/쿼리 저장(QEQS)
 소스 제어 항목의 편집, 변경 내용 및 저장을 위해 추적되어 사용자가 파일을 쉽게 관리할 수 있습니다. "체크 인"된 제어 항목을 편집하면 QEQS는 시도된 편집을 가로채서 파일을 체크 아웃할지 사용자에게 요청합니다. **도구**, **옵션** 설정에 따라 사용자는 편집하기 위해 파일을 체크 아웃해야하거나 메모리에서 복사본을 편집하고 나중에 체크 아웃 할 수 있습니다. 사용자의 **도구**, **옵션** 설정이 체크 아웃 대화 상자를 표시하고 체크 아웃하도록 설정되지 않은 경우 사용자가 편집할 때 가능하면 파일이 자동으로 체크 아웃됩니다.

#### <a name="expected-behavior"></a>예상되는 동작

- 체크 아웃 작업이 끝나면 대상 파일 및/또는 폴더가 버전 저장소에서 체크 아웃된 것으로 표시됩니다.

- 버전 저장소는 체크 아웃을 올바른 사용자에게 특성으로 합니다.

- 체크 아웃 시간과 날짜가 정확합니다(사용자 설정에 따라).

- 대상 파일 또는 폴더의 로컬 복사본을 쓸 수 있습니다.

|작업|테스트 단계|확인할 예상 결과|
|------------|----------------|--------------------------------|
|체크 인된 텍스트 파일 편집|1. 텍스트 파일이 포함된 새 프로젝트를 만듭니다.<br />2. 소스 제어에 솔루션을 추가합니다.<br />3. **설정 도구,** **옵션,** **소스 제어,** **선택을 취소하려면 디스크에서 읽기 전용으로 파일을 편집** 할 수 있습니다.<br />4. **설정 도구,** **옵션,** **소스 제어,** **체크** **아웃에 대 한 프롬프트** 체크 아웃 파일 편집 콤보 상자.<br />5. 설정 **도구,** **옵션,** **소스 제어,** 체크 **아웃에 대 한 프롬프트** 체크 아웃 파일 에서 체크 아웃 파일 은 콤보 상자를 **저장 됩니다.**<br />6. 편집기에서 텍스트 파일을 열고 파일에 새 텍스트를 입력하려고 시도합니다. 이 단계가 성공하면 다음 단계로 계속합니다.<br />7. 편집 대화 **상자체크아웃에서** **취소를** 클릭합니다. 이 단계가 성공하면 다음 단계로 계속합니다.<br />8. **설정 도구,** **옵션,** **소스 제어,** 확인 **디스크에 읽기 전용 동안 파일을 편집** 할 수 있습니다.<br />9. 편집기에서 프로젝트 파일을 열고 파일에 새 텍스트를 입력하려고 시도합니다. 이 단계가 성공하면 다음 단계로 계속합니다.<br />10. 대화 상자 **편집을 위해 체크 아웃에서** **편집을** 클릭합니다. 이 단계가 성공하면 다음 단계로 계속합니다.<br />11. 텍스트 파일을 편집하고 저장하려고 시도합니다.|`Result of step 6:`<br /><br /> 대화 상자 편집을 체크 아웃합니다.<br /><br /> `Result of step 7:`<br /><br /> 파일이 변경되지 않습니다.<br /><br /> `Result of step 9:`<br /><br /> 대화 상자 편집을 체크 아웃합니다.<br /><br /> `Result of step 10:`<br /><br /> 메모리에서 프로젝트 파일을 편집할 수 있습니다.<br /><br /> `Result of step 11:`<br /><br /> 저장시 대화 상자 저장에 체크 아웃이 나타납니다.|
|체크 인된 솔루션 파일 편집|이전 테스트에서 설명한 대로 단계를 반복하지만 텍스트 파일을 수정하는 대신 솔루션 속성을 변경하여 솔루션을 수정합니다.|이전 테스트와 동일|
|체크 인된 프로젝트 파일 편집|이전 테스트에서 설명한 대로 단계를 반복하지만 텍스트 파일을 수정하는 대신 프로젝트 속성을 변경하여 프로젝트를 수정합니다.|이전 테스트와 동일합니다.|

### <a name="case-3d-silent-check-out"></a>케이스 3D: 무음 체크아웃
 이 하위 영역은 사용자의 **도구,** **옵션,** **소스 제어 설정별로** **체크 아웃** 대화 상자가 나타나지 않는 체크 아웃 시나리오를 다룹니다.

#### <a name="expected-behavior"></a>예상되는 동작

- 체크 아웃 작업이 끝나면 대상 파일 및/또는 폴더가 버전 저장소에서 체크 아웃된 것으로 표시됩니다.

- 버전 저장소는 체크 아웃을 올바른 사용자에게 특성으로 합니다.

- 체크 아웃 시간과 날짜가 정확합니다(사용자 설정에 따라).

- 대상 파일 또는 폴더의 로컬 복사본을 쓸 수 있습니다.

|작업|테스트 단계|확인할 예상 결과|
|------------|----------------|--------------------------------|
|파일에 대한 자동 체크 아웃|1. **설정 도구,** **옵션,** **소스 제어편집에** **자동으로 파일을 체크 아웃**.<br />2. 파일로 새 프로젝트를 만듭니다.<br />3. 소스 제어에 솔루션을 추가합니다.<br />4. 파일을 체크 아웃합니다.|파일이 자동으로 체크 아웃되었습니다(UI 없음).|
|프로젝트에 대한 자동 체크 아웃|1. **설정 도구,** **옵션,** **소스 제어편집에** **자동으로 파일을 체크 아웃**.<br />2. 새 프로젝트를 만듭니다.<br />3. 소스 제어에 솔루션을 추가합니다.<br />4. 프로젝트를 확인하십시오.|파일이 자동으로 체크 아웃되었습니다(UI 없음).|

### <a name="case-3e-undo-check-out"></a>사례 3e: 체크 아웃 취소
 **체크 아웃 취소는** 파일의 체크 아웃 상태를 취소하고 파일의 변경 내용을 체크 인하지 않도록 하는 데 사용됩니다.

#### <a name="expected-behavior"></a>예상되는 동작

- 기본값은 사용자의 로컬 버전 체크 아웃 설정을 기반으로 **합니다.** 사용자가 로컬 버전을 체크 아웃하도록 선택한 경우 취소 체크 아웃의 기본값은 항상 체크 아웃된 버전으로 되돌리는 것입니다.

- 취소가 완료되면 **솔루션 탐색기의** 아이콘이 영향을 받는 파일에 대해 업데이트되고 항목이 **보류 중인 체크 인** 창에서 제거됩니다.

|작업|테스트 단계|확인할 예상 결과|
|------------|----------------|--------------------------------|
|단독으로 체크 아웃된 단일 파일의 체크 아웃 취소|1. 클라이언트 프로젝트를 만듭니다.<br />2. 소스 제어에 솔루션을 추가합니다.<br />3. 파일을 단독으로 체크 아웃합니다.<br />4. 파일을 수정합니다.<br />5. 취소 체크 아웃 **(파일,** **소스 제어,** **취소 체크 아웃).**|일반적인 예상 동작입니다.|
|공유로 체크 아웃된 단일 파일의 체크 아웃 취소|1. 클라이언트 프로젝트를 만듭니다.<br />2. 소스 제어에 솔루션을 추가합니다.<br />3. 공유된 파일을 체크 아웃합니다.<br />4. 파일을 수정합니다.<br />5. 취소 체크 아웃 **(파일,** **소스 제어,** **취소 체크 아웃).**|일반적인 예상 동작입니다.|
|프로젝트에 파일을 추가한 후 프로젝트의 체크 아웃 취소|1. 새 프로젝트를 만들고 소스 컨트롤에 추가합니다.<br />2. 프로젝트를 확인하십시오.<br />3. 프로젝트에 파일을 추가합니다.<br />4. 프로젝트의 체크 아웃을 취소합니다.|추가된 파일이 솔루션 탐색기의 프로젝트에서 제거됩니다.<br /><br /> 프로젝트가 더 이상 체크 아웃되지 않습니다.|
|프로젝트에서 파일을 삭제한 후 프로젝트의 체크 아웃 취소|1. 새 프로젝트를 만들고 소스 컨트롤에 추가합니다.<br />2. 프로젝트를 확인하십시오.<br />3. 프로젝트에서 파일을 삭제합니다.<br />4. 프로젝트의 체크 아웃을 취소합니다.|삭제된 파일은 솔루션 탐색기의 프로젝트 아래에 나타납니다.<br /><br /> 프로젝트가 더 이상 체크 아웃되지 않습니다.|
