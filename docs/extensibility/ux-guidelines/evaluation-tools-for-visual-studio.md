# <a name="evaluation-tools-for-visual-studio"></a>Visual Studio의 계산 도구
## <a name="craftsmanship-checklist-for-visual-studio"></a>비주얼 스튜디오장인 체크리스트
 이 검사 목록을 사용하여 시각적 및 상호 작용 세부 정보에 대한 사용자 환경 품질을 평가합니다.

### <a name="overview"></a>개요

- 모든 명령이 사용자에게 명령이 수행되었음을 알리는 피드백이 발생하는지 확인합니다.

- 모든 UI 요소와 컨트롤이 모든 테마와 고대비 모드에서 표시되는지 확인합니다.

- 비활성 및 활성 선택이 표준 및 고대비 모드에서 항상 차별화되는지 확인합니다.

- 포커스가 항상 표시되고 명백한지 확인합니다.

### <a name="performance"></a>성능

- 명령을 완료하는 데 1초 이상 걸리는 경우 일종의 "사용 중" 표시등이 표시되는지 확인합니다.

- 명령을 완료하는 데 10초 이상 걸리는 경우 확정(기본 설정) 또는 불확정인 명시적 진행률 표시줄이 표시되는지 확인합니다.

### <a name="ui-text"></a>UI 텍스트

- 모든 레이블이 문장 또는 제목 대/소문자이고 텍스트가 완전히 소문자가 없는지 확인합니다.

    ||올바른 코드|틀림|
    |-|-------------|---------------|
    |**명령 텍스트(모두)**|문장 대/소문자:<br /><br /> **디렉토리 이름:**|디렉토리 이름:|
    |**단추 텍스트(클라이언트)**|제목 대/소문자:<br /><br /> **[ 기본값으로 설정 ]**|기본값으로 설정|
    |**단추 텍스트(온라인)**|문장 대/소문자:<br /><br /> **[ 기본값으로 설정]**||

- 그룹 헤더와 단추를 제외한 모든 레이블이 콜론으로 끝나고 쌍을 이루는 컨트롤 앞에 오는지 확인합니다.

- 타원에서 사용자 입력 끝을 캡처하기 위해 UI를 시작하는 단추, 명령 및 명령 링크가 있는지 **확인합니다....]**.

  예제:

  - 대화 상자의 **[고급...]** 단추입니다.

  - 도구 메뉴(도구 >**옵션)의**명령 옵션은 대화 상자 자체를 실행하는 것이 명령의 의도이므로 타원을 얻지 않아야 합니다.

- UI에 업계 표준 용어를 제외하고 약어가 포함되어 있지 않은지 확인합니다. 예를 들어, OOM(메모리 부족) 및 PII(개인 식별 정보)가 필요하지만 HTML이나 TCP/IP의 철자를 입력할 필요는 없습니다.

### <a name="keyboard-access"></a>키보드 액세스

- 키보드로 각 작업을 수행할 수 있는 방법이 있는지 확인합니다. 일반적으로 이 작업은 각 컨트롤에 대한 키보드 액세스를 통해 수행되지만 일부 시각적 영역의 경우 코드 보기로 이동과 같은 해결 방법을 사용할 수 있습니다.

- 논리적 순서(왼쪽에서 오른쪽 및 위쪽에서 아래쪽으로)로 컨트롤을 탭할 수 있는지 확인합니다. 대부분의 컨트롤에 대 한 모범 사례이지만 모든 컨트롤에 이 접근 방식이 필요한 것은 아닙니다. 예를 들어 라디오 단추 컨트롤이 단일 탭 중지가 있는 그룹에 있는지 확인합니다.

- 모든 컨트롤에 레이블이 있고 각 레이블에 음수 가 있는지 확인합니다(탭에서 레이블이 지정된 컨트롤을 따를 수 있는 레이블이 지정되지 않은 컨트롤은 예외임).

- 암기 충돌이 없는지 확인합니다.

### <a name="fonts"></a>글꼴

- 모든 글꼴(얼굴, 크기, 색상)이 일관되게 사용되고 계층 구조를 유지했는지 확인합니다.

- 모든 UI 요소가 환경 글꼴 서비스를 사용하는지 확인합니다. (비주얼 [스튜디오에 대한 글꼴 및 서식 참조)](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md)

     서비스가 사용 중인지 확인하려면 **도구 > 옵션 > 글꼴 및 색상으로 이동하십시오.** 설정 드롭다운에서 환경 글꼴을 선택하고 글꼴 면을 문체적으로 다른 것(예: 해링턴 또는 코믹 샌즈)으로 변경하고 크기를 12pt로 설정합니다. 그런 후 확인을 클릭합니다. IDE를 다시 시작해야 할 수도 있지만 대부분의 UI는 즉시 변경됩니다. 다시 시작할 때에도 글꼴 변경을 선택하지 않는 영역은 환경 글꼴을 사용하지 않습니다.

- 서비스의 파생 글꼴(예: 굵게 또는 확대된 텍스트)이 환경 글꼴 크기가 변경될 때 "일반" 텍스트와 관련하여 크기와 서식을 유지했는지 확인합니다.

- 글꼴확대로 인해 클리핑 버그가 없는지 확인합니다. 잘린 글꼴은 고정 높이 컨트롤 또는 고정된 높이 컨테이너의 결과일 수 있습니다.

### <a name="dialogs"></a>대화 상자

- 대화 상자 제목이 대화 상자를 시작한 명령과 동일한지 확인합니다.

- 모든 표준 컨트롤이 운영 체제와 일치하는지 확인합니다: 배경색은 표준이며 표준 컨트롤과 다르게 표시되는 특별한 재템플릿 스타일이 없어야 합니다.

- 양식 내의 여백이 12픽셀이어야 하며 균일하고 일관되게 표시되어야 하는지 확인합니다.

- 대화 상자가 IDE 셸 또는 대화 상자가 생성된 창 가운데에 나타나는지 확인합니다.

- 유용하면 대화 상자를 재지정할 수 있어야 합니다. 크기 조정이 가능한 대화의 경우 크기를 조정할 때 대화 상자의 다른 부분이 일정하게 유지되는 동안 적절한 컨트롤의 크기를 조정해야 합니다.

- 크기 조정 가능한 대화 상자(크기, 위치, 대화 상자 컨트롤 확장 등)를 유지하는지 확인합니다.

- 제목 표시줄에 아이콘이 없는지 확인합니다.

- 제목 표시줄에 최소화 및 최대화 단추가 없는지 확인합니다.

#### <a name="dialog-operation-buttons-vs-client-only"></a>대화 상자 작업 단추(VS 클라이언트만)

- 작업 단추가 이 순서대로 되어 있는지 확인: **확인,** **취소,** **적용.**

- **확인** 및 **취소** 버튼이 표준 크기인 75x23 픽셀인지 확인합니다.

- 문자열 길이에 관계없이 **확인** 및 **취소** 단추의 크기가 동일한지 확인합니다.

- 작업 단추의 레이블에 단추를 표준보다 넓어야 하는 경우 해당 **취소** 단추의 크기가 동일한지 확인합니다.

- 단추와 연결된 컨트롤 사이에 6픽셀 패딩이 있는지 확인합니다.

- **확인** 및 **취소** 단추에 니모닉(밑줄이 그어진 문자로 정의된 액세스 키)이 없는지 확인합니다.

- 기본적으로 하나의 단추(일반적으로 **OK)에**포커스가 있는지 확인합니다.

- **Esc가** 대화 상자를 취소했는지 확인합니다.

- Enter를 처리하는 컨트롤에 포커스가 없는 경우 **Enter가** 기본 단추를 실행하는지 확인합니다.

- **확인** 및 **취소** 단추가 대화 상자의 오른쪽 하단에 있는지 확인합니다. 드문 경우지만 오른쪽 상단에 수직으로 쌓을 수 있습니다.

- 다른 단추가 대화 상자 내의 가로 정렬에 있는 경우에만 세로 구성이 사용되는지 확인합니다.

### <a name="control-standards"></a>제어 표준

#### <a name="general"></a>일반

- 가능하면 사용자 상호 작용 속도를 높이고 사용자를 안전하거나 일반적인 결과로 안내하는 좋은 기본값이 있는지 확인합니다.

- 사용자가 이전 환경을 기반으로 어떤 일이 발생할지 알 수 있도록 표준 컨트롤이 동일한 방식으로 작동하는지 확인합니다.

#### <a name="label-controls"></a>레이블 컨트롤

- 각 컨트롤에 레이블이 있고 각 레이블이 해당 컨트롤과 시각적으로 페어링되어 있는지(일반적으로 4~6픽셀 범위 내에서) 다른 컨트롤보다 해당 컨트롤에 더 가깝게 있는지 확인합니다.

- 레이블의 기준선이 왼쪽에 배치된 경우 레이블의 기준선이 입력 텍스트의 기준선과 정렬되도록 하려면 레이블이 왼쪽 모서리를 왼쪽으로 왼쪽으로 배치했는지 확인합니다.

- 여러 스택 된 레이블 및 입력 컨트롤 컨트롤컨트롤의 왼쪽에 배치 하는 경우 레이블플러시 왼쪽 및 대화 상자의 가장자리에서 동일한 거리, 결코 오른쪽으로 플러시 하지 하 고 컨트롤에서 동일한 거리 인지 확인 합니다. 그룹화를 나타내기 위해 추가 간격이 필요하지 않은 경우 쌍을 균등하게 분산해야 합니다.

#### <a name="input-controls-text-boxes-and-combo-boxes"></a>입력 컨트롤(텍스트 상자 및 콤보 상자)

- 기본 환경 글꼴을 사용할 때 텍스트 상자, 콤보 상자 및 단추의 표시 높이가 모두 23픽셀인지 확인합니다.

- 힌트 텍스트를 사용하는 경우 색상이 색상 `Environment.ControlEditHintText` 서비스를 사용하도록 설정되어 있는지 확인합니다.

- 필드가 이렇게 식별되어야 하는 필수 필드인 경우 다음을 확인합니다.

  - 배경이 `Environment.ControlEditRequiredBackground` 설정되고 전경이`Environment.ControlEditRequiredHintText`

  - 컨트롤 내에 **"필수\<>"으로** 표시되는 힌트 텍스트가 있는지

#### <a name="button-controls"></a>단추 컨트롤

- 긴 텍스트를 수용하지 않는 한 단추의 최소 크기가 75x23 픽셀인지 확인합니다.

- 단추에 3~5픽셀의 왼쪽 및 오른쪽 여백과 콘텐츠의 패딩이 있는지 확인합니다.

- **[찾아보기...]** 버튼 (또는 유사한 기능) **[...]** 대신 타원만있는 작은 사각형 버튼을 사용하는 것이 허용됩니다. 사용하는 경우 단추의 크기가 23x23인지 확인합니다.

- 대화 상자에 **[찾아보기...]** 버튼이 두 개 이상 있는 경우 단축된 버전(타원 전용 **[...])이**모두 사용되는지 확인합니다.

- 타원 **[...]** 버튼이 면모가 없는지 확인합니다. 포커스가 옆에 있는 입력 컨트롤에 있으면 한 탭이 타원 단추로 포커스를 이동해야 합니다.

- 더 많은 사용자 입력을 캡처하는 보조 UI를 시작하는 단추, 명령 및 명령 링크가 타원에서 끝나야 하는지 **확인합니다....]**.

#### <a name="hyperlinks"></a>하이퍼링크

- 하이퍼링크 컨트롤이 활성화되어 있을 때 빨간색으로 깜박이지 않는지 확인합니다. 색상 서비스가 사용되지 않음을 나타내는 표시기입니다.

- 사용되는 VS 색상이 다음과 같은지 확인합니다.

  - `Environment.ControlLinkText`

  - `Environment.ControlLinkTextHover`

  - `Environment.ControlLinkTextPressed`

- 단락에 포함되지 않는 한 하이퍼링크가 밑줄없이 파란색으로 표시되는지 확인합니다.

#### <a name="check-boxes"></a>확인란

- 확인란에 다중 줄 텍스트가 있는 경우 상자가 모든 줄에 세로로 중심이 되지 않고 첫 번째 텍스트 줄과 정렬되는지 확인합니다.

- 확인란이 항상 바이너리 선택을 나타내고 사용자를 탐색하거나 새 창이나 페이지를 열지 않는지 확인합니다.

- 확인란에 입력 컨트롤과 관련된 옵션이 표시되면 플러시가 플러시 왼쪽에 위치하고 해당 컨트롤 아래에 매우 가깝게 배치되어 있는지 확인하여 해당 상태를 나타냅니다.

- 대화 상자 또는 페이지의 전체 내용을 활성화하는 수단으로 확인란이 사용되지 **않았는지** 확인합니다.

#### <a name="group-boxes"></a>그룹 상자

- 대화 상자에 대화 상자의 전체 내용을 포함 하는 단일 그룹 상자가 없는지 확인 합니다.

- 각 그룹 상자 내에 두 개 이상의 컨트롤이 있는지 확인합니다.

- 대화 상자에 두 개 이상의 그룹 상자가 있는 경우는 거의 없습니다.

- 중첩된 그룹 상자가 없는지 확인합니다.

### <a name="icons"></a>아이콘

- 어두운 테마에서 아이콘이 올바르게 반전된지 확인합니다.

- 모든 아이콘이 핵심 개념을 기반으로 하는지 확인합니다.

- 각 아이콘이 뚜렷하고 인식하기 쉽고 두 개 이상의 개념이 포함되어 있지 않은지 확인합니다(상태 수정자/언어 제외).

- 기본 아이콘이 공간 가운데에 나타나는지 확인합니다.

- 고대비 모드에서 모든 아이콘이 읽기 쉬운 지 확인합니다.

- 사용된 색상이 색상 사용 표준과 일치하는지 확인합니다.

- 아이콘 주위에 후광(테두리)이 없는지 확인합니다. (있는 경우 후광은 인접 UI의 배경색과 일치해야 합니다.)

### <a name="touch-enabled-ui"></a>터치 지원 UI

- 인터랙티브 컨트롤이 쉽게 만질 수 있을 만큼 충분히 큰지 확인 - 최소 **23x23 픽셀** 크기

- 가장 자주 사용되는 컨트롤의 크기가 **40x40픽셀** 이상인지 확인합니다.
