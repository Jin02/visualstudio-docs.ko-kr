# <a name="ca2227-collection-properties-should-be-read-only"></a>CA2227: 컬렉션 속성은 읽기 전용이어야 합니다.

|||
|-|-|
|TypeName|CollectionPropertiesShouldBeReadOnly|
|CheckId|CA2227|
|범주|마이크로소프트.사용|
|주요 변경 내용|주요 변경|

## <a name="cause"></a>원인

외부에서 볼 수 있는 쓰기 가능한 속성은 <xref:System.Collections.ICollection?displayProperty=fullName>을 구현하는 형식입니다. 이 규칙은 배열, 인덱서('Item'라는 이름의 속성), 변경할 수 없는 컬렉션, readonly 컬렉션 및 권한 집합을 무시합니다.

## <a name="rule-description"></a>규칙 설명

쓰기 가능한 컬렉션 속성을 사용하면 컬렉션을 완전히 다른 컬렉션으로 바꿀 수 있습니다. 읽기 전용 속성은 컬렉션의 교체를 중지하지만 개별 멤버를 설정할 수 있습니다. 컬렉션을 대체하는 것이 목표인 경우 기본 설정 디자인 패턴은 컬렉션에서 모든 요소를 제거하는 방법과 컬렉션을 다시 채우는 메서드를 포함하는 것입니다. 이 <xref:System.Collections.ArrayList.Clear%2A> 패턴의 예는 <xref:System.Collections.ArrayList.AddRange%2A> <xref:System.Collections.ArrayList?displayProperty=fullName> 클래스의 메서드 및 메서드를 참조하십시오.

이진 및 XML 직렬화 모두 컬렉션인 읽기 전용 속성을 지원합니다. <xref:System.Xml.Serialization.XmlSerializer?displayProperty=fullName> 클래스에는 직렬화할 수 있도록 구현하고 <xref:System.Collections.ICollection> <xref:System.Collections.IEnumerable?displayProperty=fullName> 구현하기 위해 형식에 대한 특정 요구 사항이 있습니다.

## <a name="how-to-fix-violations"></a>위반 문제를 해결하는 방법

이 규칙의 위반을 해결하려면 속성을 읽기 전용으로 만드십시오. 디자인에 필요한 경우 컬렉션을 지우고 다시 채우는 메서드를 추가합니다.

## <a name="when-to-suppress-warnings"></a>경고를 억제하는 경우

속성이 [DTO(데이터 전송 개체)](/previous-versions/msp-n-p/ff649585(v=pandp.10)) 클래스의 일부인 경우 경고를 표시하지 않습니다.

그렇지 않으면 이 규칙에서 경고를 표시하지 마십시오.
