# <a name="c6011"></a>C6011
경고 C6011: NULL 포인터 역참조 \<이름 >

 이 경고는 null 포인터 역참조 되는 것을 나타냅니다. 포인터 값이 잘못된 경우 결과가 정의되지 않습니다.

## <a name="example"></a>예제
 다음 코드에서는 호출 malloc 사용할 수 있는 메모리가 부족 한 경우 null을 반환할 수 있으므로이 경고를 생성 합니다.

```
#include <malloc.h>

void f( )
{
  char *p = ( char * ) malloc( 10 );
  *p = '\0';

  // code ...
 free( p );
}
```

 이 경고를 해결 하려면 다음 코드와 같이 null 값에 대 한 포인터를 검사 합니다.

```
#include <malloc.h>
void f( )
{
  char *p = ( char * )malloc ( 10 );
  if ( p )
  {
    *p = '\0';
    // code ...

    free( p );
  }
}
```

 매개 변수를 가진 매개 변수를 역참조 하기 전에 사전 조건에서 Null 속성을 사용 하 여 주석이 지정 된 함수 내에서 메모리를 할당 해야 합니다. Null 포인터를 역참조 하려고 하기 때문에 다음 코드에서는 경고 C6011 (`pc`) 첫 번째 메모리를 할당 하지 않고 함수 내에서:

```
#include <sal.h>
using namespace vc_attributes;
void f([Pre(Null=Yes)] char* pc)
{
  *pc='\0'; // warning C6011 - pc is null
  // code ...
}
```

 무료 및 malloc의 사용 경우 메모리 누수 및 예외를 기준으로 많은 문제 이러한 종류의 누수 및 문제 모두 예외를 방지 하려면에서 제공 하는 메커니즘을 사용 합니다 C++ 표준 템플릿 라이브러리 (STL). 여기에 포함 됩니다 [shared_ptr](/cpp/standard-library/shared-ptr-class)하십시오 [unique_ptr](/cpp/standard-library/unique-ptr-class), 및 [벡터](/cpp/standard-library/vector). 자세한 내용은 [스마트 포인터](/cpp/cpp/smart-pointers-modern-cpp) 하 고 [ C++ 표준 라이브러리](/cpp/standard-library/cpp-standard-library-reference)합니다.
