# <a name="c6255"></a>C6255
경고 C6255: _alloca는 스택 오버플로 예외를 발생 시켜 실패를 나타냅니다. 대신 _malloca 사용을 고려 하세요.

 이 경고는 로컬 예외 처리 외부에서 `_alloca`에 대 한 호출이 검색 되었음을 나타냅니다. `_alloca`은 오류 발생 시 스택 오버플로 예외를 발생 시킬 수 있으므로 예외 처리기의 보호 된 범위 내에서 항상 호출 해야 합니다. 가능 하면 @no__t를 사용 하는 대신-0을 사용 하는 것이 좋습니다.-1은 `_alloca`의 보다 안전한 버전인 @no__t.

## <a name="example"></a>예제
 다음 코드는 `_alloca`에서 예외를 생성할 수 있기 때문에이 경고를 생성 합니다.

```
#include <windows.h>

void f( )
{
  void *p = _alloca(10);
  // code ...
}
```

 이 경고를 해결 하려면 `_malloca`을 사용 하 고 다음 코드와 같이 예외 처리기를 추가 합니다.

```
#include <windows.h>
#include <malloc.h>

void f( )
{
  void *p;
  int errcode;
  __try
  {
    p = _malloca(10);
    // code...
    _freea(p);
  }
  __except( (GetExceptionCode() == STATUS_STACK_OVERFLOW ) ?
              EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH )
  {
    errcode = _resetstkoflw();
    // code ...
  }
}
```
