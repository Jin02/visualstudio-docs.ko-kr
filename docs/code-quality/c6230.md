# <a name="c6230"></a>C6230

경고 C6230: 의미 체계가 다른 정수 형식 간의 암시적 캐스트: 부울 컨텍스트에서 HRESULT를 사용 하 여

이 경고는 완전 `HRESULT` 는 컨텍스트에서 사용 되는 부울 결과가 필요한 경우, 같은 `if` 문입니다. 이 테스트는 잘못 된 결과가 발생할 수 있습니다. 예를 들어, 일반적인 성공 값 `HRESULT` (`S_OK`) 부울으로 테스트할 경우 false입니다.

## <a name="example"></a>예제

다음 코드에서는이 경고를 생성합니다.

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (hr)
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

대부분의 경우는 [SUCCEEDED](/windows/desktop/api/winerror/nf-winerror-succeeded) 또는 [실패](/windows/desktop/api/winerror/nf-winerror-failed) 매크로의 값을 테스트 하려면 사용 해야는 `HRESULT`합니다. 이 경고를 해결 하려면 다음 코드를 사용 합니다.

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (SUCCEEDED(hr))
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

이 경고는 `SCODE` 형식으로 처리 됩니다는 `HRESULT`합니다.

사용 `malloc` 고 `free` (및 동적 메모리 Api 관련) 메모리 누수 및 예외를 발생 하는 많은 문제에 있습니다. 제공 하는 포인터와 컨테이너 클래스를 사용 하 여 이러한 종류의 누수 및 예외 문제를 방지 하려면는 C++ 표준 라이브러리입니다. 여기에 포함 됩니다 [shared_ptr](/cpp/standard-library/shared-ptr-class)하십시오 [unique_ptr](/cpp/standard-library/unique-ptr-class), 및 [벡터](/cpp/standard-library/vector). 자세한 내용은 [스마트 포인터](/cpp/cpp/smart-pointers-modern-cpp) 하 고 [ C++ 표준 라이브러리](/cpp/standard-library/cpp-standard-library-reference)합니다.
