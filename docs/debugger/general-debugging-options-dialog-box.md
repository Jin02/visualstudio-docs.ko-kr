# <a name="general-debugging-options"></a>일반 디버깅 옵션

Visual Studio 디버거 옵션을 설정하려면 **도구** > **옵션을**선택하고 **디버깅** 에서 **일반** 옵션 옆의 상자를 선택하거나 선택 취소합니다. **도구** > **가져오기 및 내보내기 설정** > 재설정으로 모든 기본**설정을 복원할**수 있습니다. 설정의 하위 집합을 재설정하려면 테스트할 설정을 변경하기 전에 **설정 가져오기 마법사를** 사용하여 설정을 저장한 다음 나중에 저장된 설정을 가져옵니다.

다음 **일반** 옵션을 설정할 수 있습니다.

**모든 중단점을 삭제하기 전에 질문하기:** **모든 중단점 삭제** 명령을 완료하기 전에 확인이 필요합니다.

**하나의 프로세스가 중단될 때 모든 프로세스를 중단:** 동시에 중단이 발생할 때 디버거가 연결된 모든 프로세스를 중단합니다.

**예외가 AppDomain 또는 관리/네이티브 경계를 교차하는 경우 중단**: 관리되는 또는 혼합 모드 디버깅에서 공통 언어 런타임은 다음 조건이 true일 때 응용 프로그램 도메인 경계 또는 관리/네이티브 경계를 교차하는 예외를 catch할 수 있습니다.

1. 네이티브 코드가 COM Interop를 사용하여 관리 코드를 호출하고 관리 코드가 예외를 throw하는 경우. [COM Interop 소개를](/dotnet/articles/visual-basic/programming-guide/com-interop/introduction-to-com-interop)참조하십시오.

2. 애플리케이션 도메인 1에서 실행되는 관리 코드가 애플리케이션 도메인 2의 관리 코드를 호출하고 애플리케이션 도메인 2의 코드가 예외를 throw하는 경우. [애플리케이션 도메인으로 프로그래밍](/dotnet/articles/framework/app-domains/index)을 참조하세요.

3. 코드가 리플렉션을 사용하여 함수를 호출하면 해당 함수가 예외를 throw합니다. [리플렉션](/dotnet/framework/reflection-and-codedom/reflection)을 참조하십시오.

조건 2 및 3에서는 예외가 공용 언어 `mscorlib` 런타임이 아닌 관리 코드에서 catch되는 경우가 있습니다. 이 옵션은 `mscorlib`를 통해 catch된 예외에서의 중단에 영향을 미치지 않습니다.

**주소 수준 디버깅 사용:** 주소 수준(디스어셈블리 창, 레지스터 창 및 주소 중단점)에서 디버깅을 위한 고급 기능을 **활성화합니다.** **Disassembly**

- **소스를 사용할 수 없는 경우 디스어셈블리 표시:** 소스를 사용할 수 없는 코드를 디버깅할 때 **디스어셈블리** 해제 창이 자동으로 표시됩니다.

**중단점 필터 사용**: 중단점에 필터를 설정하여 특정 프로세스, 스레드 또는 컴퓨터에만 영향을 줄 수 있습니다.

**새 예외 도우미 사용:** 예외 도우미를 대체하는 예외 도우미를 사용하도록 합니다. (예외 도우미는 Visual Studio 2017에서 부터 지원됩니다.)

> [!NOTE]
> 관리 코드의 경우 이 옵션은 이전에 **예외 도우미 활성화라고** 불렸습니다.

**내 코드 만 사용**: 디버거는 시스템 코드 및 최적화되어 있거나 디버깅 기호가없는 다른 코드를 무시하고 사용자 코드 ("내 코드")만 표시하고 단계합니다.

- **실행 시 사용자 코드가 없는 경우 경고(관리만 가능)**: 내 코드만 사용하도록 디버깅을 시작하면 사용자 코드("내 코드")가 없는 경우 이 옵션을 경고합니다.

**.NET 프레임워크 소스 스테핑 사용**: 디버거가 .NET Framework 소스로 단계별로 들어갈 수 있도록 합니다. 이 옵션을 사용하면 내 코드만 자동으로 비활성화됩니다. .NET 프레임워크 기호는 캐시 위치에 다운로드됩니다. **옵션** 대화 상자, **디버깅** 범주, 기호 페이지를 사용 하 고 캐시 위치를 변경 **합니다.**

**속성 및 연산자 단계(관리만)**: 디버거가 관리 코드의 속성 및 연산자로 침입하지 못하도록 합니다.

**속성 평가 및 기타 암시적 함수 호출 사용**: 변수 창 및 **QuickWatch** 대화 상자에서 속성 및 암시적 함수 호출의 자동 평가를 켭니다.

- **변수 창의 개체에 대한 문자열 변환 함수 호출(C# 및 JavaScript만)**: 변수 창에서 개체를 평가할 때 암시적 문자열 변환 호출을 실행합니다. 결과는 형식 이름 대신 문자열로 표시됩니다. C# 코드에서 디버깅하는 동안에만 적용됩니다. 이 설정은 디버거디스플레이 특성에 의해 재정의될 수 [있습니다(디버거디스플레이 특성 사용](../debugger/using-the-debuggerdisplay-attribute.md)참조).

**소스 서버 지원 사용**: Visual Studio 디버거에게 SrcSrv ()`srcsrv.dll`프로토콜을 구현하는 원본 서버에서 소스 파일을 가져옵니다. Team Foundation Server와 Debugging Tools for Windows는 프로토콜을 구현하는 두 가지 소스 서버입니다. SrcSrv 설정에 대한 자세한 내용은 [SrcSrv](/windows-hardware/drivers/debugger/srcsrv) 설명서를 참조하십시오. 또한 [기호(.pdb) 및 소스 파일 지정을](../debugger/specify-symbol-dot-pdb-and-source-files-in-the-visual-studio-debugger.md)참조하십시오.

> [!IMPORTANT]
> *.pdb* 파일을 읽으면 파일에서 임의의 코드가 실행될 수 있으므로 반드시 서버를 신뢰해야 합니다.

- **출력 창에 소스 서버 진단 메시지 인쇄**: 원본 서버 지원이 활성화되면 이 설정은 진단 표시를 켭니다.

- **부분 신뢰 어셈블리에 대해 원본 서버 허용(관리만 해당)**: 원본 서버 지원이 활성화된 경우 이 설정은 부분 신뢰 어셈블리에 대한 원본을 검색하지 않는 기본 동작을 재정의합니다.

- **항상 메시지 없이 신뢰할 수 없는 원본 서버 명령을 실행:** 원본 서버 지원이 활성화 된 경우 이 설정은 신뢰할 수 없는 명령을 실행할 때 프롬프트의 기본 동작을 재정의합니다.

**소스 링크 지원 사용**: Visual Studio 디버거에게 소스 링크 정보가 포함된 *.pdb* 파일에 대한 원본 파일을 다운로드하도록 지시합니다. 소스 링크에 대한 자세한 내용은 [소스 링크 사양을](/dotnet/standard/library-guidance/sourcelink)참조하십시오.

> [!IMPORTANT]
> 소스 링크는 http 또는 https를 사용하여 파일을 다운로드하므로 *.pdb* 파일을 신뢰해야 합니다.

- **모든 소스 링크 요청에 대한 Git 자격 증명 관리자 인증으로 대체:** 소스 링크 지원이 활성화되고 소스 링크 요청이 인증에 실패하면 Visual Studio에서 Git 자격 증명 관리자를 호출합니다.

**중단점 및 현재 문에 대한 전체 줄을 강조 표시 (C ++ 만)**: 디버거가 중단점 또는 현재 문을 강조 표시 할 때 전체 줄을 강조 표시합니다.

**원본 버전과 정확히 일치하도록 원본 파일 필요**: 원본 파일이 디버깅 중인 실행 파일을 빌드하는 데 사용되는 소스 코드 의 버전과 일치하는지 확인하기 위해 디버거에게 알려줍니다. 버전이 일치하지 않으면 일치하는 소스를 찾으라는 메시지가 표시됩니다. 일치하는 소스가 없으면 디버깅하는 동안 소스 코드가 표시되지 않습니다.

**모든 출력 창 텍스트를 즉시 창으로 리디렉션:** 일반적으로 **출력** 창에 나타나는 모든 디버거 메시지를 **즉시** 창으로 보냅니다.

**변수 창에서 객체의 원시 구조 표시**: 모든 개체 구조 뷰 사용자 지정을 끕니다. 뷰 사용자 지정에 대한 자세한 내용은 [관리되는 개체의 사용자 지정 보기 만들기를](../debugger/create-custom-views-of-managed-objects.md)참조하십시오.

**모듈 로드시 JIT 최적화 억제(관리만)**: 디버거가 연결된 동안 모듈이 로드되고 JIT가 컴파일될 때 관리코드의 JIT 최적화를 비활성화합니다. 최적화를 해제하면 성능이 저하될 수 있지만 일부 문제를 보다 쉽게 디버깅할 수 있습니다. 내 코드만 기능을 사용하는 경우 JIT 최적화를 해제하면 사용자가 작성하지 않은 코드가 사용자 코드("내 코드")로 표시될 수 있습니다. 자세한 내용은 [JIT 최적화 및 디버깅](../debugger/jit-optimization-and-debugging.md)을 참조하십시오.

**ASP.NET(크롬, 마이크로소프트 에지 및 IE)에 대한 JavaScript 디버깅 사용 :** ASP.NET 앱에 대한 스크립트 디버거를 활성화합니다. Chrome에서 처음 사용할 때는 설치한 Chrome 확장 프로그램을 사용하려면 브라우저에 로그인해야 할 수 있습니다. 이 옵션을 사용하지 않도록 설정하여 레거시 동작으로 되돌리세요.

**UWP 자바 스크립트 애플 리 케이 션에 대 한 가장자리 개발자 도구를 사용 (실험)**: 마이크로소프트 가장자리에서 UWP 자바 스크립트 애플 리 케이 션에 대 한 개발자 도구를 사용할 수 있습니다.

**ASP.NET 레거시 크롬 자바 스크립트 디버거 사용**: ASP.NET 애플 리케이션에 대한 레거시 크롬 자바 스크립트 스크립트 디버거를 활성화합니다. Chrome에서 처음 사용할 때는 설치한 Chrome 확장 프로그램을 사용하려면 브라우저에 로그인해야 할 수 있습니다.

**비주얼 스튜디오를 관리자로 실행할 때 실험적인 방법을 사용하여 Chrome JavaScript 디버깅을 실행합니다.**

**로드 dll 내보내기(기본 만)**: dLL 내보내기 테이블을 로드합니다. Windows 메시지, Windows 프로시저(WindowProcs), COM 개체, 마샬링 또는 기호가 없는 dll을 사용하여 작업하는 경우 dll 내보내기 테이블의 기호 정보가 유용할 수 있지만, dll 내보내기 정보를 읽으면 일부 오버헤드가 발생합니다. 따라서 이 기능은 기본적으로 해제되어 있습니다.

dll의 내보내기 테이블에서 사용할 수 있는 기호를 확인하려면 `dumpbin /exports`를 사용하세요. 모든 32비트 시스템 dll에 기호를 사용할 수 있습니다. `dumpbin /exports` 출력을 읽으면 영숫자가 아닌 문자를 포함하여 정확한 함수 이름을 확인할 수 있습니다. 이 방법은 함수에 중단점을 설정하는 데 유용합니다. dll 내보내기 테이블에 있는 함수 이름은 디버거에서 일부가 잘린 상태로 나타날 수 있습니다. 호출은 현재 함수(가장 안쪽에 중첩된)가 맨 위에 표시되어 호출한 순서로 나열됩니다. 자세한 내용은 [dumpbin /exports](/cpp/build/reference/dash-exports)를 참조하십시오.

**병렬 스택 다이어그램 상향식 표시**: 병렬 스택 창에 스택이 표시되는 방향을 **제어합니다.**

**작성된 데이터가 값을 변경하지 않은 경우 GPU 메모리 액세스 예외 무시**: 데이터가 변경되지 않은 경우 디버깅 중에 검색된 경합 조건을 무시합니다. 자세한 내용은 [GPU 코드 디버깅을](../debugger/debugging-gpu-code.md)참조하십시오.

**관리되는 호환성 모드 사용**: 기본 디버깅 엔진을 레거시 버전으로 대체하여 다음 시나리오를 활성화합니다.

- 자체 표현식 계산기(C++/CLI 포함)를 제공하는 C#, Visual Basic 또는 F#이 아닌 .NET 언어를 사용하고 있습니다.

- 혼합 모드 디버깅 중에 C++ 프로젝트에 대해 편집 및 계속을 사용하도록 설정하려고 합니다.

> [!NOTE]
> 관리되는 호환성 모드를 선택하면 기본 디버깅 엔진에서만 구현되는 일부 기능이 비활성화됩니다. 레거시 디버깅 엔진은 Visual Studio 2012에서 교체되었습니다.

**레거시 C# 및 VB 식 계산기 사용**: 디버거는 Visual Studio 2015 Roslyn 기반 식 계산기 대신 Visual Studio 2013 C# 또는 Visual Basic 식 계산기를 사용합니다.

**잠재적으로 안전하지 않은 프로세스에 대해 사용자 지정 디버거 시각화 도우미를 사용할 때 경고(관리만 해당)**: Visual Studio는 안전하지 않은 코드를 실행할 수 있으므로 디버깅된 프로세스에서 코드를 실행하는 사용자 지정 디버거 시각화 도우미를 사용할 때 경고합니다.

**Windows 디버그 힙 할당자 활성화(기본 전용)**: Windows 디버그 힙을 활성화하여 힙 진단을 개선할 수 있습니다. 이 옵션을 사용하면 디버깅 성능에 영향을 줍니다.

**XAML에 대한 UI 디버깅 도구 사용**: 지원되는 프로젝트 유형인**F5(디버깅)를**시작할 때 라이브 비주얼 트리 및 라이브 속성 탐색 창이 나타납니다. 자세한 내용은 [디버깅하는 동안 XAML 속성 검사를](../xaml-tools/inspect-xaml-properties-while-debugging.md)참조하십시오.

- **라이브 비주얼 트리에서 선택한 요소 미리 보기**: 컨텍스트가 선택된 XAML 요소도 라이브 비주얼 **트리** 창에서 선택됩니다.

- **응용 프로그램에서 런타임 도구 표시**: 디버깅 중인 XAML 응용 프로그램의 기본 창에 있는 도구 모음에 **라이브 시각적 트리** 명령을 표시합니다. 이 옵션은 Visual Studio 2015 업데이트 2에서 도입되었습니다.

- **XAML 핫 로드 사용**: 앱을 실행할 때 XAML 코드와 함께 XAML 핫 다시로드 기능을 사용할 수 있습니다. (이 기능은 이전에 "XAML 편집 및 계속"이라고 불렸습니다.

::: moniker range=">= vs-2019" 
- **내 XAML 만 사용 :** Visual Studio 2019 버전 16.4에서 시작하여 **기본적으로 라이브 비주얼 트리에는** 사용자 코드로 분류된 XAML만 표시됩니다. 이 옵션을 사용하지 않도록 설정하면 생성된 모든 XAML 코드가 도구에 표시됩니다.

- **요소를 선택한 경우 선택 모드 끄기** Visual Studio 2019 버전 16.4부터 인앱 도구 모음 요소 선택기**버튼(선택 활성화)은**요소를 선택하면 꺼져 있습니다. 이 옵션을 사용하지 않도록 설정하면 인앱 도구 모음 단추를 다시 클릭할 때까지 요소 선택이 유지됩니다.
::: moniker-end

**디버깅 하는 동안 진단 도구 사용:** 디버깅 하는 동안 **진단 도구** 창이 나타납니다.

**디버깅 하는 동안 경과 시간 PerfTip 표시**: 코드 창디버깅 할 때 지정된 메서드 호출의 경과 시간을 표시합니다.

**편집 및 계속 사용 :** 디버깅 하는 동안 편집 및 계속 기능을 활성화합니다.

- **네이티브 편집 및 계속 사용 :** 네이티브 C++ 코드를 디버깅하는 동안 편집 및 계속 기능을 사용할 수 있습니다. 자세한 내용은 [편집 및 계속(C++)을](../debugger/edit-and-continue-visual-cpp.md)참조하십시오.

- **계속 시 변경 내용 적용(네이티브만 해당)**: Visual Studio는 중단 상태에서 프로세스를 계속할 때 수행한 미해결 코드 변경 사항을 자동으로 컴파일하고 적용합니다. 이 옵션을 선택하지 않으면 **디버그** 메뉴에서 **코드 변경 적용** 항목을 사용하여 변경 내용을 적용하도록 선택할 수 있습니다.

- **부실 코드에 대한 경고(기본 만)**: 오래된 코드에 대한 경고를 가져옵니다.

**디버깅 하는 동안 편집기에서 클릭 으로 실행 실행 단추**표시 : 이 옵션을 선택하면 디버깅 하는 동안 클릭 [실행](../debugger/debugger-feature-tour.md#run-to-a-point-in-your-code-quickly-using-the-mouse) 버튼이 표시 됩니다.

**디버깅이 중지될 때 콘솔을 자동으로 닫는다:** Visual Studio에 디버깅 세션이 끝날 때 콘솔을 닫도록 지시합니다.

::: moniker range=">= vs-2019"
**빠른 식 평가 활성화(관리전용)**: 디버거가 간단한 속성 및 메서드의 실행을 시뮬레이션하여 더 빠른 평가를 시도할 수 있도록 합니다.
::: moniker-end

## <a name="options-available-in-older-versions-of-visual-studio"></a>이전 버전의 Visual Studio에서 사용할 수 있는 옵션

이전 버전의 Visual Studio를 사용하는 경우 몇 가지 추가 옵션이 있을 수 있습니다.

**예외 도우미 사용**: 관리 코드의 경우 예외 도우미를 사용하도록 설정합니다. Visual Studio 2017에서 시작하여 예외 도우미가 예외 도우미를 대체했습니다.

**처리되지 않은 예외에서 호출 스택 해제**: **호출 스택** 창이 처리되지 않은 예외가 발생하기 전에 호출 스택을 지점으로 롤백하도록 합니다.

**시작 시 기호가 없는 경우 경고(기본 전용)**: 디버거에 기호 정보가 없는 프로그램을 디버깅할 때 경고 대화 상자가 표시됩니다.

실행 중 **스크립트 디버깅이 비활성화된 경우 경고:** 스크립트 디버깅을 사용하지 않도록 설정한 경우 디버거를 시작할 때 경고 대화 상자를 표시합니다.

**기본 호환성 모드 사용**: 이 옵션을 선택하면 디버거는 새 네이티브 디버거 대신 Visual Studio 2010 네이티브 디버거를 사용합니다.

- 새 디버깅 엔진은 .NET C++ 식 평가를 지원하지 않으므로 .NET C++ 코드를 디버깅할 때 이 옵션을 사용합니다. 하지만 기본 호환성 모드를 사용할 경우 현재 디버거 구현에 따라 많은 기능을 작동할 수 없습니다. 예를 들어 레거시 엔진에는 Visual Studio 2015 `std::string` 프로젝트와 같은 기본 제공 형식에 대한 많은 시각화 도우미가 없습니다.   이러한 경우 최적의 디버깅 환경을 위해 Visual Studio 2013 프로젝트를 사용합니다.
